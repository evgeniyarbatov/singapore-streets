# Architecture

Singapore Streets turns OpenStreetMap (OSM) highway data into a categorized catalog of Singapore street names with map polylines. The pipeline is linear: extract → clean → categorize → publish.

## Pipeline overview

```
Geofabrik OSM PBF
       │
       ▼
  make osm / make city          Clip to Singapore (osmconvert + osmium)
       │
       ▼
  extract_streets.py            Named highways → polylines + name list
       │
       ▼
  make clean                    format-address → invalid-address → street-names
       │
       ▼
  categorize_streets.py         Rules → LLM → human review overrides
       │
       ▼
  create-dataset.py             Inner join names + categories + polylines
       │
       ▼
  dataset/singapore-streets.csv Published dataset (Kaggle)
       │
       ▼
  build_site.py                 Static catalog → site/dist (GitHub Pages)
```

Each stage writes artifacts under `data/` or `dataset/`. Intermediate reject logs go to `filtered/`.

## Design principles

**OSM as source of truth for geometry.** Street polylines come from OSM `highway` ways and named relations with `name` (or alias) tags. There is no separate geocoding step — the same extract that produces names also produces coordinates. What counts as a street (include / exclude / gray zone) is defined in [street-definition.md](street-definition.md).

**Cleaning is a filter chain, not a single pass.** Three small stdin/stdout scripts run in sequence during `make clean`. Each step has a narrow job (normalize spelling, drop junk, keep only street-shaped names). Rejected lines go to an explicit `--reject-log` path (default under `filtered/`) for inspection, and a curated `data/allowlist.txt` lets confirmed edge cases skip the building/slash filters.

**Stable taxonomy over free-form labels.** Categories are defined in `data/taxonomy.yaml` — currently 12 primary categories plus `uncategorized`. Classification prefers deterministic rules, then a constrained local LLM (Ollama).

**Resumable categorization.** `categorize_streets.py` appends to `data/street_categories.csv` and skips streets already present. Re-runs are safe after adding new names.

## Directory layout

```
singapore-streets/
├── osm/                    # Downloaded and clipped OSM files (gitignored)
│   ├── singapore.poly      # Boundary polygon for clipping
│   └── singapore.osm       # Singapore extract (XML)
├── data/                   # Pipeline working data (mostly gitignored)
│   ├── taxonomy.yaml       # Stable category definitions + rules (tracked)
│   ├── allowlist.txt       # Confirmed edge-case names that bypass cleaning filters (tracked)
│   ├── osm-streets.csv     # name, polyline, osm_source, aliases
│   ├── street-names.txt    # One name per line (cleaned)
│   ├── review-queue.csv    # Duplicate/missing polylines from extract_streets.py
│   ├── canonical-streets.csv    # canonical_name, display_name, aliases
│   ├── street_categories.csv    # Categorized names (generated)
│   └── category-stats.json # Generated by category_report.py
├── filtered/               # Reject logs from cleaning (gitignored)
├── dataset/                # Publishable output
│   └── singapore-streets.csv
├── prompts/
│   └── categorize-v1.md    # Versioned LLM prompt template
├── site/
│   ├── static/             # app.js, style.css (source)
│   └── dist/               # Generated static site (committed for Pages)
├── scripts/                # Pipeline scripts (see scripts.md)
├── tests/                  # Unit tests per script
├── Makefile                # Orchestration targets
└── docs/                   # This documentation
```

## Data artifacts

### `data/osm-streets.csv`

| Column | Description |
|--------|-------------|
| `name` | Street name as tagged in OSM (falls back through `name:en`, `name:ms`, `name:zh`, `alt_name`, `old_name` when no plain `name` tag exists) |
| `polyline` | Encoded polyline string (Google algorithm) |
| `osm_source` | `highway_name`, `name_pattern`, or `relation_name` |
| `aliases` | Pipe-separated alternate names merged from the tags above |

Multiple OSM ways with the same name are merged into one polyline when endpoints are within ~25 m. Named relations (e.g. expressways tagged as `type=route, route=road`) stitch their member ways' geometry the same way.

### `data/street-names.txt`

Plain text, one street name per line. This is the canonical name list after cleaning. Row count is the project's street total (~4,400).

### `data/review-queue.csv`

Written by `extract_streets.py`. Columns: `issue_type` (`duplicate_polyline` or `missing_polyline`), `names`, `polyline`. Lets duplicate geometries and streets with no resolvable polyline be triaged instead of scrolling past on stderr.

### `data/canonical-streets.csv`

Written by `canonical_streets.py`. One row per logical street: `canonical_name` (direction suffix stripped), `display_name` (preferred form), `aliases` (pipe-separated direction variants, e.g. `Foo Road East`/`Foo Road West`).

### `data/street_categories.csv`

| Column | Description |
|--------|-------------|
| `street_name` | Matches `street-names.txt` |
| `primary_category` | Taxonomy id (e.g. `malay_archipelago`) |
| `category` | Human-readable display name |
| `tags` | Pipe-separated secondary tags (e.g. `person\|estate`) |
| `source` | `rule`, `llm`, `llm_timeout`, `llm_error` |
| `prompt_version` | Prompt file version when LLM was used |
| `model` | Ollama model name |
| `legacy_category` | Original free-form label, if migrated |

### `dataset/singapore-streets.csv`

Final publishable dataset — inner join of `street-names.txt`, `street_categories.csv`, and `data/osm-streets.csv`:

| Column | Description |
|--------|-------------|
| `street_name` | Official-style name |
| `category` | Display category from taxonomy |
| `polyline` | Map geometry |

Streets without a category are excluded (inner join).

## Categorization flow

```
street-names.txt ──► classify_by_rules() ──► matched? ──► write CSV
                               │
                               │ no match
                               ▼
                         Ollama + prompt
                         (JSON: primary_category, tags)
                               │
                               ▼
                         write CSV (resumable)
                               │
                               ▼
                    categories-override.csv
                    (manual corrections, git-tracked)
```

Rules live in `data/taxonomy.yaml` under `rules:` and `colonial_surnames:`. The shared loader is `scripts/taxonomy.py`. Overrides in `data/categories-override.csv` replace any prior label for that street.

## Static site

`scripts/build_site.py` joins the publishable dataset with category tags and OSM/canonical aliases into gzipped `site/dist/data/streets.json.gz`, plus a single-page Leaflet UI (`index.html`, `app.js`, `style.css`). District, etymology, and personal notes render when those fields exist later.

`site/dist` is committed so GitHub Actions can deploy without regenerating pipeline data (mostly gitignored). Push to `main` runs `.github/workflows/pages.yml`.

## Testing

Tests live in `tests/` and mirror `scripts/`. They cover extraction/merge logic, each cleaning filter, categorization (mocked Ollama), taxonomy rules, dataset creation, and the site builder. Run with `make test`.
